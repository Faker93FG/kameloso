MinimalAuthentication

kameloso plugins common awareness

Implements triggering of queued events in a plugin module.

Most of the time a plugin doesn't require a full UserAwareness; only those that need looking up users outside of the current event do. The persistency service allows for plugins to just read the information from the dialect.defs.IRCUser embedded in the event directly, and that's often enough.

General rule: if a plugin doesn't access kameloso.plugins.common.core.IRCPluginState.users, it's probably going to be enough with only MinimalAuthentication.
